【ステップ3：図・シミュ・画像の差し込み計画（設計書）】

このチャットの**直前のメッセージ**（あなたがステップ2で出力したレビュー用解説）と、ステップ1のJSON（会話の前の方にある）、およびユーザーが送った問題画像を参照し、**各設問に置く図・シミュ・画像の差し込み計画**を以下の形式で出力してください。

■ 設計書の形式（設問ごと）

## 設問 (X)(y) の図・シミュ
| 項目 | 内容 |
|------|------|
| 種類 | SVG / Canvasグラフ / エネルギー棒グラフ(HTML+CSS) / シミュ(p5.js想定：プレースホルダのみ) / 画像(プレースホルダ) / 図なし |
| 目的 | 何を視覚化するか（1行） |
| 要素 | 図に含めるべき要素のリスト（例：電源V₀、スイッチS1/S2、抵抗R、コンデンサーCA/CB、電流Iの向き） |
| 状態 | どのスイッチの開閉に対応した状態か（該当する場合） |
| 補足 | 凡例・軸ラベル・単位など |
| HTML差し込み位置 | ステップ4のHTMLで、どの card のどの位置に置くか（原則：h3直後） |
| ID/参照名 | sim-embed の id、canvas id、図のaria-label案など（衝突しない一意な名前） |

■ 設計方針
・回路図・力の図・状況図 → インライン **SVG**（viewBox で座標系、線・円・テキストで描画）。クラス名は circuit-svg や diagram-svg。
・時系列グラフ（I-t, v-t 等）→ **Canvas**。&lt;canvas id="chart-xxx" width="520" height="280"&gt; + インライン script。
・割合・エネルギー配分の比較 → **エネルギー棒グラフ**。.energy-bar-wrap + .bar-outer + .bar-fill。色は .fill-ca, .fill-cb, .fill-joule 等。
・動きのあるシミュ（運動・振動）→ **p5.js想定**。ただしステップ4では実装コードは書かず、&lt;div id="sim-xxx" class="sim-embed"&gt;&lt;/div&gt; の**プレースホルダだけ**置く（後から差し込む）。
・問題画像・参考図の貼り付け → **画像プレースホルダ**。&lt;figure class="fig-inline"&gt;&lt;img src="【画像パス】"&gt;... の形で置く（実ファイルは作れない）。
・図は設問の h3 の直後（解説文の前）に配置する。

すべての設問について、図を置く場合はこの形式で設計書を出し、図が不要な設問は「図なし」と明記すること。
